# issue issue 46 に関連して、agentがハルシネーションして、現実を計測するという対応をせず、対応に失敗した。人力で現実を計測する場合の手法を洗い出して整理する #56
[issues #56](https://github.com/cat2151/quartz-transformer-mmlabc/issues/56)

## 解決策

詳細は [56-solution.md](56-solution.md) を参照してください。

### 概要

issue #46において、AIエージェントが実際の問題を計測せずにハルシネーション（幻覚）による解決策を提案し、対応に失敗しました。このドキュメントでは、人力で現実を計測する場合の手法を体系的に整理しています。

### 主要な内容

1. **ブラウザベースの計測手法**
   - 開発者ツールの活用（Console, Network, Elements, Sources）
   - コンソールでの動的な確認
   - スーパーリロードの活用

2. **自動テストを使った計測手法**
   - Playwrightによるブラウザテスト
   - ユニットテストによるロジック検証

3. **デバッグツールとロギング手法**
   - 戦略的なコンソールログの配置
   - エラーハンドリングの強化

4. **タイミングと実行順序の計測手法**
   - スクリプトの読み込み順序の確認
   - 非同期処理のデバッグ

5. **実践的なワークフロー**
   - 問題発生時の標準調査手順（8ステップ）
   - 計測結果の記録フォーマット

6. **ハルシネーション防止のための原則**
   - 推測より計測
   - エラーメッセージを軽視しない
   - 実際のブラウザで確認
   - 自動テストで検証
   - データを記録する
